# R2 Storage — Project Context

## What This Is
Google Drive-like UI for Cloudflare R2. Next.js 16 App Router deployed on Cloudflare Workers via @opennextjs/cloudflare.

## Architecture
- **Frontend**: React 19 client components + shadcn/ui + TailwindCSS v4 + TanStack Query + Zustand
- **Backend**: Next.js API Route Handlers accessing R2 bindings via `getCloudflareContext()`
- **Auth**: Cloudflare Access (Zero Trust) — no app-level auth code, CF Access is the gateway
- **Deploy**: Cloudflare Workers Builds from GitHub, auto-deploys on push to main

## Key Patterns
- R2 buckets auto-discovered at runtime via `discoverBuckets()` in `src/lib/r2.ts`
- File keys use URL encoding in API paths (`/api/buckets/[bucket]/object/[...key]`)
- Share metadata stored as `.r2-storage/shares/{id}.json` inside each R2 bucket
- Folders are zero-byte objects with trailing `/`
- Move = copy + delete (R2 has no native rename)
- Multipart upload for files >= 95MB (Workers have 100MB request limit)
- ANY new R2 mutation handler MUST schedule indexing via `ctx.waitUntil(indexOnWrite/Delete(...))` after the mutation succeeds. See `src/lib/indexing.ts`. Forgetting this = silent search staleness bug.
- ANY new R2 mutation handler MUST ALSO call `ctx.waitUntil(recordActivity(...))` to append to the D1 activity log. See `src/lib/audit.ts`. Forgetting this = silent audit gap.
- ANY new R2 mutation handler MUST ALSO `ctx.waitUntil(invalidateListCache(env, bucketName))` after the mutation succeeds. See `src/lib/listing-cache.ts`. Forgetting this = stale folder listings.
- All Workers AI calls go through `src/lib/ai.ts` (`embed` / `chatStream`), never `env.AI.run` directly. This ensures traffic flows through the AI Gateway.

## File Layout
- `src/app/api/` — Route handlers (all R2 operations)
- `src/components/` — UI components (layout, files, upload, dialogs, shadcn ui/)
- `src/hooks/` — TanStack Query hooks (use-file-list, use-server-config)
- `src/stores/` — Zustand (app-store, upload-store, selection-store)
- `src/lib/` — Utilities (api-client, r2 helpers, file-utils, encoding)
- `src/types/` — TypeScript interfaces

## Build & Deploy
- `pnpm build` = `next build` (for local/CI)
- `pnpm deploy` = `opennextjs-cloudflare build && deploy`
- `wrangler.jsonc` has `build.command` for Workers Builds
- `.dev.vars` for local env (gitignored)
- `cloudflare-env.d.ts` generated by `pnpm cf-typegen`

## R2 Buckets
Configured in `wrangler.jsonc` under `r2_buckets`. Currently: `WEBSITE_ASSETS`, `EXPLORER_TEST`.
